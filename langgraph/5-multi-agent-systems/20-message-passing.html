Message Passing in LangGraph#

Message passing in LangGraph is the primary mechanism by which agents, nodes, and tools communicate and coordinate during execution. It enables multi-agent collaboration, state evolution, memory construction, reasoning loops, and decision-making in complex LLM workflows.

1. Core Idea#

LangGraph models communication as state updates. Every node receives the global state, produces partial updates, and these updates are merged back into the shared state.

Message passing = controlled mutation of shared state

2. Conceptual Model#

Node A → update(state) → Global State → Node B → update(state) → ...

Nodes do not talk directly. They communicate indirectly by reading and writing messages in the state.

3. Message Structure#

A message is typically stored in the state as a list of structured objects.

from typing import TypedDict, List
from langchain.schema import BaseMessage

class State(TypedDict):
    messages: List[BaseMessage]

Each message contains:

Field

Description

role

system / user / assistant / tool

content

textual content

metadata

tool name, timestamps, ids

4. Sending Messages (Node Output)#

Nodes emit messages by returning updated state fragments.

from langchain.schema import AIMessage

def agent_node(state: State):
    response = AIMessage(content="I will search for information.")
    return {"messages": state["messages"] + [response]}

LangGraph merges this partial update into the global state.

5. Receiving Messages (Node Input)#

Every node receives the full current message history.

def tool_node(state: State):
    last_msg = state["messages"][-1].content

This enables:

  • conversation context

  • agent coordination

  • tool instruction extraction

6. Multi-Agent Message Passing#

Each agent writes messages into the shared state.

Supervisor → "Find market data"
Worker A → "Collected financials"
Worker B → "Computed growth rates"
Supervisor → "Generate report"

All of this appears as ordered messages in state["messages"].

7. Routing via Messages#

Decisions are often made by inspecting messages.

def router(state: State):
    text = state["messages"][-1].content.lower()
    if "search" in text:
        return "search_node"
    return "final_node"

Messages therefore control execution flow.

8. Message Reducers & Accumulation#

LangGraph uses reducers to merge concurrent message updates.

from langgraph.graph import StateGraph

def message_reducer(old, new):
    return old + new

builder.add_state_reducer("messages", message_reducer)

This guarantees:

  • no lost messages

  • deterministic ordering

  • concurrency safety

9. Message Passing in Cycles#

In cyclic graphs, messages evolve over iterations.

Reason → Act → Observe → (messages updated) → Reason → ...

This enables:

  • reflection

  • self-correction

  • iterative planning

11. Minimal Example#

from langgraph.graph import StateGraph, END
from typing import TypedDict, List
from langchain.schema import HumanMessage, AIMessage

class State(TypedDict):
    messages: List

def chatbot(state):
    user = state["messages"][-1].content
    reply = AIMessage(content=f"You said: {user}")
    return {"messages": state["messages"] + [reply]}

builder = StateGraph(State)
builder.add_node("chat", chatbot)
builder.set_entry_point("chat")
builder.add_edge("chat", END)

graph = builder.compile()

result = graph.invoke({"messages": [HumanMessage(content="Hello")]})

Demonstration#

from typing import TypedDict, List
from langchain_classic.schema import HumanMessage, AIMessage
from langgraph.graph import StateGraph, END

# -------------------------
# 1. Define Shared State
# -------------------------
class State(TypedDict):
    messages: List

# -------------------------
# 2. Define Nodes
# -------------------------
def agent(state: State):
    user_text = state["messages"][-1].content
    reply = AIMessage(content=f"Agent received: {user_text}")
    return {"messages": state["messages"] + [reply]}

def router(state: State):
    text = state["messages"][-1].content.lower()
    if "tool" in text:
        return "tool"
    return "final"

def tool(state: State):
    msg = AIMessage(content="Tool executed successfully.")
    return {"messages": state["messages"] + [msg]}

def final(state: State):
    msg = AIMessage(content="Conversation complete.")
    return {"messages": state["messages"] + [msg]}

# -------------------------
# 3. Build Graph
# -------------------------
builder = StateGraph(State)

builder.add_node("agent", agent)
builder.add_node("tool", tool)
builder.add_node("final", final)

builder.set_entry_point("agent")

builder.add_conditional_edges("agent", router, {
    "tool": "tool",
    "final": "final"
})

builder.add_edge("tool", "final")
builder.add_edge("final", END)

graph = builder.compile()

# -------------------------
# 4. Run
# -------------------------
result = graph.invoke({"messages": [HumanMessage(content="Please use a tool")]})

# -------------------------
# 5. Inspect Message Passing
# -------------------------
for m in result["messages"]:
    print(f"{m.type.upper()}: {m.content}")

AI: Agent received: Please use a tool
AI: Tool executed successfully.
AI: Conversation complete.
